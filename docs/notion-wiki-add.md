🔴 현재 파일에서 빈약하거나 추가가 필요한 부분
1. 취미/사이드 콘텐츠 섹션이 아예 없음
지금 말씀하신 네이버 블로그, Medium, YouTube 채널이 Wiki에 전혀 반영되어 있지 않아요. 이걸 담을 "창작/콘텐츠 활동" 섹션이나 "About (비개발)" 섹션 하나가 없습니다. AskOosu에서 "우수는 어떤 사람이에요?" 류 질문에 답할 때 쓸 수 있는 인간적인 레이어가 부족해요.
2. Lab 섹션 내용이 거의 비어있음 (우선순위 3 - 25번)
CSS Experiments, JavaScript Projects의 구체적 목록이 없고, Lab이 무엇을 하는 공간인지 설명은 있어도 실제 콘텐츠가 빈칸이에요.
3. 연락/협업 섹션이 없음
FAQ 8번("협업하거나 연락하려면?")은 있는데, 실제로 Wiki 안에 Connect/Contact 섹션이 없어요. 현재 구직 상태, 어떤 협업을 원하는지, 어떤 방식으로 연락을 원하는지가 없습니다.
4. 코드 리뷰 방식 (TODO로 방치)
일하는 방식 테이블의 코드 리뷰 항목이 TODO로 비어있어요. AskOosu가 "어떻게 협업하나요?" 질문에 답할 때 빈칸입니다.
5. "포트폴리오를 왜 대화형으로 만들었어요?" (FAQ 7번)
AskOosu의 정체성 자체를 설명하는 가장 중요한 FAQ인데, 우선순위 1에 올려놓고 아직 본인 언어로 작성이 안 된 상태예요. RAG가 이 질문을 받았을 때 답이 없는 상황입니다.
6. AI 코드 검증/실력 증명 관련 두 FAQ
AI Usage FAQ의 핵심 두 질문(실력 증명, 검증 루틴)이 모두 TODO입니다. 실제로 채용 담당자가 가장 많이 물을 수 있는 질문들이에요.

🟡 말씀하신 취미 자료들을 Wiki에 넣는다면
각각 어느 정도 비중으로 올리면 좋을지 정리해드릴게요.

특히 YouTube 채널은 Lab 섹션에 넣기 딱 좋아요. "비주얼 실험, 그래디언트 스크린세이버"는 UI 감각과 연결되고, 개발자로서 "만들어서 배포했다"는 실행력을 보여주는 작은 증거가 됩니다.
Medium 글이 개발/UX/브랜딩 관련이라면 Writing 서브섹션으로 올려서 "우수는 어떻게 생각하고 글로 정리하는 사람인가"를 보여주는 데 쓸 수 있어요.

📌 4. AI Usage FAQ — TODO 두 항목 보완
markdown#### Q. AI로 코딩하면 본인 실력은 어떻게 증명하나요?

AI를 쓴다는 것과 개발 실력이 있다는 것은 서로 다른 차원의 이야기라고 생각합니다. AI는 코드를 빠르게 생성해주지만, 어떤 구조로 설계할지, 어떤 기술 선택이 이 상황에 맞는지, 코드가 왜 작동하는지 혹은 왜 안 되는지를 판단하는 건 여전히 사람의 몫입니다. 우수는 AI 없이 바닐라 JS로 포트폴리오를 구현한 경험, 코딩 테스트 문제를 직접 먼저 풀어보는 습관, 실서비스 배포와 디버깅 경험을 통해 개발 실력의 기반을 쌓고 있습니다. AI를 잘 쓰는 것 자체가 실력이라고 생각하지만, 그 전제는 코드를 읽고 판단할 수 있어야 한다는 것입니다.

#### Q. AI를 쓸 때 어떤 기준으로 검증하나요?

AI가 작성한 코드에는 반드시 주석 설명을 함께 요청합니다. 코드 흐름을 직접 읽으며 로직을 파악하고, 실행해서 동작을 확인합니다. 이해되지 않는 부분은 콘솔 출력이나 중간 단위 테스트로 검증하고, 같은 실수를 반복하지 않기 위해 어떤 부분에서 오류가 났는지 기록해두는 방식을 씁니다. 코딩 테스트 문제는 AI 없이 직접 먼저 풀어보고 이후에 AI로 검토하는 방식으로 판단력을 유지합니다.
