# Product Build Loop

## Purpose

Defines the loop for turning ideas into portfolio projects or service-like demos: problem, user scenario, scope, prototype, integration, review, documentation, and iteration.

## Operating Rules

- Define the user problem before features.
- Add AI only when it connects to the real UX flow.
- Avoid fake progress, unverified metrics, and inflated savings/source-count claims.
- Design loading, failure, empty, and success states together.
- After building, document facts, story, decision log, postmortem, FAQ, and guardrail.

## Answer Usage

Describe Oosu’s workflow as connecting problem framing, validation, and documentation, not merely building many things fast.

## Do Not Say

- More features always make a better portfolio.
- AI features are always good.
- Speed matters more than completeness.
