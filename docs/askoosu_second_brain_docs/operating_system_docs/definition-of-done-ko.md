# Definition of Done

## 목적

우수의 프로젝트에서 완료란 기능이 존재하는 상태가 아니라, 사용자가 목적을 이해하고 실패 상황까지 납득할 수 있는 상태입니다.

## 운영 규칙

- 첫 화면에서 제품 목적이 이해되어야 한다.
- 핵심 CTA와 사용자 행동이 명확해야 한다.
- loading/error/empty/success 상태가 있어야 한다.
- AI 답변은 source, confidence, fallback 정책과 연결되어야 한다.
- FAQ ID 또는 RAG source chunk가 있어야 한다.
- lint/typecheck와 변경 파일/테스트 방법 정리가 필요하다.

## 답변 활용 방향

“완성” 질문에는 배포 여부보다 사용자 이해, 신뢰, fallback, 문서화, 검증을 기준으로 답합니다.

## 피해야 할 표현

- 배포되었으면 완료입니다.
- AI 답변이 나오면 기능은 완성입니다.
- demo니까 과장 copy를 써도 괜찮습니다.

## 연결 질문

- 우수는 언제 프로젝트를 완성으로 보나요?
- AskOosu에서 완료 기준은 무엇인가요?
- AI 기능을 넣을 때 무엇을 확인하나요?

## 검토 메모

도구 선호도, 프로젝트 구조, 공개 포지셔닝이 바뀌면 이 문서를 함께 갱신해야 합니다.
