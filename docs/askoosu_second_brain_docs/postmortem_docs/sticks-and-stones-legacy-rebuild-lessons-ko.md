# Postmortem: Sticks & Stones Legacy Rebuild Lessons

## 요약

Sticks & Stones는 실제 서비스 웹사이트의 레거시 구조를 이해하고 브랜드 경험을 보존하면서 Vite + TypeScript로 현대화한 프로젝트입니다.

## 잘 된 점

- 실제 클라이언트/서비스 맥락
- 레거시 WordPress/PHP/CSS/JS 구조 이해
- 브랜드 identity 보존과 기술 기반 현대화

## 어려웠던 점 / 리스크

- 레거시 작업은 기술보다 맥락 파악이 어려움
- 무엇을 보존하고 무엇을 바꿀지 판단 필요
- 검증되지 않은 성능/성과 수치 claim 금지

## 배운 점

- 실제 서비스 리빌드는 새로 만드는 능력보다 기존 맥락을 해치지 않고 개선하는 판단력이 중요하다.
- 구조적 개선은 검증 가능한 범위에서만 말한다.

## 답변 활용 방향

이 문서는 사용자가 프로젝트의 한계, 회고, 다시 만든다면 바꿀 점을 물을 때 사용합니다. 완벽한 척하지 말고, 확인된 범위에서 배운 점과 다음 개선 방향을 설명합니다.

## 피해야 할 표현

- 이 프로젝트는 완벽했습니다.
- 모든 기능이 production 수준입니다.
- 검증되지 않은 성과 수치가 있습니다.

## Confidence

Medium-high. 기존 위키, 프로젝트 설명, 대화에서 확인된 설계 방향에 근거합니다.
