# Postmortem: Portfoli-Oh! JSON Chatbot Limitations

## Summary

Portfoli-Oh! used GSAP, Three.js, Lottie, custom cursor, highlighter interactions, and a JSON chatbot. It showed technical exploration, but became less user-centered and the JSON chatbot became hard to maintain as content grew.

## What Went Well

- Useful product signal was created.
- The project exposed a real constraint or learning moment.
- The lesson became input for later AskOosu/AI portfolio decisions.

## What Was Hard / Risk

- The project revealed limitations in scope, trust, architecture, or maintainability.
- Some claims must remain cautious unless verified.
- The next version should document boundaries and failure states more clearly.

## Lessons

- Use the project as a learning milestone, not as an overclaimed production case.
- Connect the lesson back to AskOosu answer quality, trust, and product judgment.
